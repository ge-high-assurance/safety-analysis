Module Validation

module Validation: sig .. end
Validation checks are provided to assist the end-user in building component library and models created following the modeling constructs defined for fault tree. Checks include those for the component library only, for the model only, and for the model relative to the library.

Validation checks for component library only:

  • checkLibrary_nonEmptyFaults : All components must support at least one flow, otherwise there is nothing to analyze.
  • checkLibrary_componentUnique : A library of components is a list of components, such that no two components have the same name.
  • checkLibrary_formulasMakeSense : Check that the formulas make sense.
  • checkLibrary_disjointInputFlowsandBasicEvents : Check that the names used for input flows, and basic events are disjoint.
  • checkLibrary_allOutputFaultsHaveFormulas : All flows have a formula defined.
  • checkLibrary_listsAreConsistentLengths : Check that the lists are all of consistent lengths.
Validation checks for component model only:

  • checkModel_inputFlowUnique : Check that input flows have exactly 1 instance connected to them.
  • checkModel_instanceNameUnique : All instance names are disjoint.
Validation checks for component model relative to a library:

  • checkModel_validConnections : The connection information is a list of pairs consisting of an input flow and an instance.
  • checkModel_cnameInstanceIsDefinedInLibrary : All c_names correspond to actual components in our component library.
  • checkModel_exposureOfBasicIsDefinedInLibrary : Elements of the form (a, x) in exposures of an instance make sense.

val checkLibrary_nonEmptyFaults : Modeling.component list -> (string, string) Core.Std._result
A function that checks that all components in a library support at least one flow. Otherwise there is nothing to analyze.
val checkLibrary_componentUnique : Modeling.component list -> (string, string) Core.Std._result
A function that checks that a library of components is a list of components, such that no two components have the same name.
val checkLibrary_formulasMakeSense : Modeling.component list -> (string, string) Core.Std._result
A function that checks that the formulas of all library components make sense. All formulas include formula of the form F c;b or F c, where c is a subset of the variables in the union of input_flows, basic events, and c, where c can be defined as a local variable within the list of formulas.
val checkLibrary_disjointInputFlowsandBasicEvents : Modeling.component list -> (string, string) Core.Std._result
A function that checks that the library component names used for input flows and basic events are disjoint.
val checkLibrary_allOutputFaultsHaveFormulas : Modeling.component list -> (string, string) Core.Std._result
A function that checks that all library component flows have a formula defined.
val checkLibrary_listsAreConsistentLengths : Modeling.component list -> (string, string) Core.Std._result
A function that checks that the lists within each library component are all of consistent lengths, e.g., the length of basic_events is the same as the length of event_info. Also, the length of faults x output_flows is the length of formulas.
val checkModel_inputFlowUnique : Modeling.model -> (string, string) Core.Std._result
A function that checks that in the connections of a model, input flows have at most 1 instance connected to them.
val checkModel_instanceNameUnique : Modeling.model -> (string, string) Core.Std._result
A function that checks that in a model all instance names are disjoint.
val checkModel_validConnections : Modeling.model ->
       Modeling.component Core.Std.List.t -> (string, string) Core.Std._result
A function that checks that the model connections, relative to a library, is a list of pairs consisting of an input flow and an instance.
val checkModel_cnameInstanceIsDefinedInLibrary : Modeling.model ->
       Modeling.component list -> (string, string) Core.Std._result
A function that checks that all in a model, relative to a library, the model c_names correspond to actual components in the component library.
val checkModel_exposureOfBasicIsDefinedInLibrary : Modeling.model ->
       Modeling.component list -> (string, string) Core.Std._result
A function that checks that the model elements, relative to a library, of the form (a, x) in exposures of an instance make sense, i.e., there is a basic event named a in the component being instantiated.
